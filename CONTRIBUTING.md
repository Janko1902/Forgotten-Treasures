# Branches and PRs

When creating branches, we will follow the generic naming convention:

* `feature/FT-x`
* `fix/FT-x-y` → `y` refers to the related feature if possible. If you can’t find it or there’s no relation, just omit it.
* `epic/FT-x`

`x` being the corresponding issue number.

When creating a PR:

* **Always** mark it as a **Draft** until it is completely finished and ready for review.
* Optionally, prefix the title with `WIP:` if the PR is still in progress.
